My idea for this project is to do a Tamagotchi styled interaction "game". It's not really a full game but I included just the necessary amount of interaction needed for fluid animation sequence and user engagement. I personally never played Tamagotchi so I based my project on the few things I've seen. To explain what a Tamagotchi is (if you don't know), it is this little egg-shaped "consol" that has a small screen on it and when you start the game, an egg hatches and you have three buttons to press on to interact with the little 8-bit sprite, the whole premise is to try to keep it alive and let it grow. 
Interactions I have included are: User agreement pop-up, insert-name pop-up, three buttons that allows you to either feed, play, or put the tamagotchi to sleep, changing sprites to show slight animation and also making the game feel more alive, detecting button spam sequence that forces a pop-up telling the user to slow down (pop-up has a slight animation), audio (user can turn it on or off button), moving background image. 
I used mainly Javascript but there are a few CSS animations such as the moving background image, sprite blinking, spam pop-up shaking. There is also one gsap. 
I found the background image on this free background image website and I also found the audio on another free audio website. The fonts (2) are from Google Fonts. Everything else I drew by hand or coded.
